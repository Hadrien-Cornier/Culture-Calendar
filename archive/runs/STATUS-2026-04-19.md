## Decisions worth recording

### visual_arts modelled on `concert`, not `movie` (T1.2)

The template lifts `concert`'s field set (`artists`, `works`, `duration_minutes`, `venue`, `description`) rather than `movie`'s (director / runtime / rating certificate). Rationale:

- Visual-arts events on NowPlayingAustin are mostly exhibitions and openings — temporal "performance"-like windows with an ensemble of featured artists, not single-director works.
- Keeps the UI-side card template close to concerts (multi-artist marquee line, work list), which means minimal front-end work in `docs/script.js`.
- Avoids misusing `movie` fields (e.g. a certificate rating makes no sense for a gallery opening).

### Art-critic rating branch as a third tier in EventProcessor (T1.6)

`_get_visual_arts_rating()` is a new branch alongside the existing film-critic and music-critic prompts, rather than falling back to the generic one. Rationale:

- The rubric differs: conceptual rigor, material craftsmanship, curatorial framing, and institutional context matter more than e.g. acting or tempo.
- Keeps ratings distribution meaningful across event categories — mixing all categories into one prompt was the root cause of flat ratings we've flagged in "Known Issues" of `CLAUDE.md`.
- No change to downstream callers; the processor's public signature is unchanged.

### v12d promoted over v12i, despite equal composite score (T3.3 → T3.4)

Both v12d and v12i scored composite 8.6. The tie-breaker per spec is `critique` (both 8) and then aria-count / implementation simplicity. v12d won on:

- Native `<details>` element → aria-count 11 (highest across variants) with no custom keyboard handling for expand/collapse.
- Zero-row resting chrome — the shortest filter bar at rest, which directly addresses the morning-checklist "visibly shorter" acceptance.
- Three 44 px tap-target rules → strongest mobile a11y signal in the batch.

### Libra Books is actually Livra Books (T0.1 → T2.2)

The queue named the venue `Libra` but the real Austin indie bookstore is `Livra` (`livrabooks.com`). The scraper targets the correct domain; internal identifiers keep the "Libra" spelling to honour the queue's ID. If the site is ever renamed, rename the scraper class + config key in one pass.

### T2.1 self-BLOCKED is the correct outcome

T0.2's audit proved that Alienated Majesty only publishes book clubs — no `/events`, `/news`, or `/calendar` page with artist-talk / reading / signing / exhibit content exists. Extending the scraper without a source would have been speculative. T2.1 blocked with `needs-source: no-public-events-page`, which the run contract treats as non-fatal. This is the correct call; do **not** re-open T2.1 until a public AM events URL is produced.

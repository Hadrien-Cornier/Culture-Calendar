## T5.2 recovery — what went wrong and how it was fixed

**Original blocker at 05:16**: `persona_critique.py` imports `check_live_site.py` via `importlib.util.spec_from_file_location` + `exec_module` to share its `MOBILE_VIEWPORT` constants and `_resolve_browser_executable` helper. Under Python 3.13, `@dataclass` decoration calls `sys.modules.get(cls.__module__).__dict__` to resolve `KW_ONLY` sentinel types. Without registering the module in `sys.modules` before `exec_module`, this lookup returns `None` and `@dataclass` raises `AttributeError: 'NoneType' object has no attribute '__dict__'`.

**Fix (commit `90f9e51`)**:

```python
def _load_check_live_site_module() -> Any:
    module_name = "_check_live_site"
    cached = sys.modules.get(module_name)
    if cached is not None:
        return cached
    here = Path(__file__).resolve().parent / "check_live_site.py"
    spec = importlib.util.spec_from_file_location(module_name, here)
    assert spec is not None and spec.loader is not None
    mod = importlib.util.module_from_spec(spec)
    sys.modules[module_name] = mod  # must happen BEFORE exec_module
    try:
        spec.loader.exec_module(mod)
    except Exception:
        sys.modules.pop(module_name, None)
        raise
    return mod
```

Regression test `test_load_check_live_site_module_registers_in_sys_modules` exercises the exact import path and asserts `sys.modules['_check_live_site']` is populated post-call.

### Suggested follow-ups from the LLM council

The qualitative persona critiques in `docs/PERSONAS.md` surfaced four issues worth the next run or a hot-fix:

1. **Mobile title overflow** — very long uppercase titles (e.g., "THE NEW YORKER WEEKLY SHORT STORY CLUB (FREE COPIES, FIRST HOUR FOR READING) - THIS WEEK'S STORY") dominate 375px viewports. Proposed fix: `display: -webkit-box; -webkit-line-clamp: 2; -webkit-box-orient: vertical; overflow: hidden` on card titles.
2. **Expand affordance invisible** — review-reader persona notes the `▶` triangle isn't labeled as "tap to expand". Proposed fix: add a "Read review ↓" text link beneath the subtitle, OR default-open the #1 ranked pick.
3. **Search-user wants filter chips back** (tension with T1.1a removal) — their user story is "jazz concerts this weekend" without typing. Consider a lightweight category-chip row BELOW the search bar (not the old drawer) OR a permanent "Show filters" expando next to the search input.
4. **Venue addresses missing** — logistics-user passes structurally but notes the card shows only a cryptic venue abbreviation ("AFS"), no street / neighborhood. Proposed fix: surface a neighborhood or address line in the card subtitle.

None of these are regressions from this run — they're pre-existing or user-brief-driven design decisions surfaced by the LLM council. Worth a design discussion before implementation.
